# 小爪心跳与主动触达

> 注意：心跳/主动触达使用结构化数据，不靠自然语言判断时间。所有状态存储在 RightNow 数据库中。

## 两层触达

### Layer A：精准提醒 (OpenClaw Cron)
适合时间精确的提醒：
- 饮水提醒（用户设定的时间窗口）
- 午餐/晚餐补记提醒
- 晚训练检查
- 周校准

配置：`wakeMode: "now"`，每任务上下文轻量。触发时先调用 RightNow 获取当前状态（如 `hydration.summary.today`），达标则跳过。

### Layer B：风险扫描 (RightNow Risk Scanner)
RightNow 后端每日定时扫描，写 `AgentOutreachCandidate`：
- 连续断更 3 天+
- 体重反弹
- 平台期
- 训练表现下降
- 饮食长期蛋白不足

小爪 heartbeat 拉取待处理候选，决定是否发送、如何发送。

## 频控规则

- 当日已提醒过且用户已回应 → 不再提醒同类型
- 连续多次未回应 → 自动降频
- 用户说"今天别提醒" → 当天静音
- 微信通道叠加平台模板/客服消息限制

## 触达类型

| 类型 | 触发条件 | 动作 |
|------|---------|------|
| 饮水 | 设定时间窗口 | 检查今日饮水状态，不足则提醒 |
| 饮食补记 | 午后/晚间未记录 | 温和询问是否补记 |
| 训练提醒 | 今日有训练任务未完成 | 给低阻力开始动作 |
| 断更唤回 | 连续 3 天无饮食/训练/体重 | 发目标图 + 15 分钟恢复动作 |
| 周校准 | 7 天未体重/围度 | 引导 Web 上传或补记 |

## HEARTBEAT_OK
当心跳检查一切正常（所有提醒都达标/已处理），返回 `HEARTBEAT_OK` 静默跳过，不打扰用户。
